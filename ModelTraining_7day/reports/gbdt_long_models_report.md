# NSW 未来 7 天半小时电价预测模型评测报告

## 任务说明

目标是预测 NSW 区域未来 7 天每日 48 个半小时电价，即每个样本输出 `7 × 48 = 336` 个 RRP 价格点。

当前训练数据来自：

- 每日 48 个半小时历史电价：`2015To2026Data.csv`
- 每日最高温：`max_temps_2015_2026.csv`
- 每日最低温：`min_temps_2015_2026.csv`
- 平均温度：由 `(最高温 + 最低温) / 2` 计算得到

## 数据与切分

- 样本格式：过去 28 天历史信息预测未来 7 天电价。
- 训练集和测试集按时间顺序切分，避免未来信息泄漏。
- 训练样本数：`3253`，日期范围 `2015-01-28` 到 `2023-12-24`。
- 测试样本数：`814`，日期范围 `2023-12-25` 到 `2026-04-21`。

## 模型设置

### Persistence

直接复制最近 7 天的电价曲线作为未来 7 天预测。它不使用温度，是最低基准线。

### ARIMA

采用槽位级 ARIMA：对 48 个半小时槽位分别训练一个 ARIMA(1,0,1) 模型，然后组合成未来 7 天 × 48 点预测。该模型只使用历史电价。

### LightGBM / XGBoost

采用长表建模方式：每个样本展开成 336 行，每行预测一个「预测日 + 半小时槽位」的价格。

使用特征包括：

- 最近 7 天同一半小时槽位历史电价
- 同一槽位过去 28 天电价的均值、标准差、最小值、最大值
- 最近一天电价的均值、最小值、最大值、标准差
- 历史最高温、最低温、平均温度统计
- 未来预测日最高温、最低温、平均温度
- 预测日和半小时槽位的周期编码

注意：回测中未来温度使用真实观测温度；真实预测时需要替换为天气预报。

## 结论

1. LightGBM 和 XGBoost 的 MAE/RMSE 明显优于 Persistence，说明最高温、最低温、平均温度与历史电价特征能够提供有效预测信号。
2. ARIMA 也优于 Persistence，但略弱于 LightGBM/XGBoost，说明纯历史价格的线性时间序列模型能提供基准能力，但对非线性、尖峰和天气影响刻画不足。
3. LightGBM 当前整体 MAE 最低，XGBoost 非常接近；两者都可作为后续交易策略优化的主要候选模型。
4. MAPE 在电价任务中非常不稳定，因为 RRP 会接近 0 或出现负值；建议主要参考 MAE、RMSE 和 sMAPE。
5. RMSE 仍然较高，说明极端价格尖峰仍难预测。后续应加入需求、AEMO 预测、机组停运、LOR/PASA、新闻事件等特征。

## 加入新闻特征后的直接多步预测结果

本轮继续采用**直接多步预测模式**。模型仍然预测未来 7 天 × 48 个半小时电价，但在原有历史电价、最高温、最低温、平均温度特征基础上，额外加入历史新闻特征。

新闻数据来源：`NSW_EPNews/CollectedData/Classified news/allnews2026.csv`。

为避免未来信息泄漏，新闻特征只使用预测日前已经发生的新闻，不使用未来 7 天预测窗口内的新闻。每个样本加入的新闻特征包括：

- 过去 28 天新闻总数
- 过去 28 天 Level 1 / Level 2 / Level 3 / Unknown 新闻数量
- 最近 7 天新闻总数
- 最近 7 天 Level 1 / Level 2 / Level 3 / Unknown 新闻数量

### 新闻特征结果解读

当前这一版新闻特征没有提升模型效果，LightGBM_news 和 XGBoost_news 的 MAE/RMSE 均略差于不含新闻的对应模型。原因可能包括：

1. 当前新闻特征过粗，只统计新闻数量和 Level 数量，没有保留新闻具体语义、地区、事件类型和影响时段。
2. `allnews2026.csv` 的 2025/2026 新闻由新的抓取和 DeepSeek 分类生成，和 2015-2024 历史新闻的密度、口径可能不完全一致。
3. 电价尖峰通常由供需、机组停运、约束、AEMO 预测和天气共同驱动，简单新闻计数不足以解释尖峰。
4. 新闻发布时间与电价影响存在滞后和事件窗口，简单的 7 天/28 天计数没有显式建模这种时间影响。

因此，当前正式候选模型仍建议优先使用不含新闻的 LightGBM / XGBoost 版本。新闻特征后续更适合改成结构化事件特征，例如：

- 仅保留 NSW/NEM 相关 Level 1 和 Level 2 事件
- 提取事件类型：outage、LOR、PASA、system strength、transmission constraint、weather、price spike 等
- 按事件影响日期而不是新闻发布日期对齐
- 对每日新闻做语义聚合或 embedding，而不是只用数量统计
